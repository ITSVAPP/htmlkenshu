# 調べ方と AI プロンプト

「やりたいこと別サンプル」で見つからないときは、自分で調べます。**調べる力**は実務でいちばん大事なスキルです。

## 検索のコツ

- **やりたいことをそのまま日本語で**打つ：例「CSS 文字 中央 寄せる」「HTML 画像 リンクにする」
- **エラーはそのままコピペ**して検索する（英語のままでも OK）
- 公式情報を優先：検索語に **`MDN`** を足すと信頼できるページが出やすい（例「flex 横並び MDN」）
- 一度で出なければ、**言葉を変えて**何度か試す

## おすすめの調べ先

- [MDN Web Docs（日本語）](https://developer.mozilla.org/ja/) … HTML / CSS / JavaScript の公式リファレンス
- 検索エンジン（Google など）
- 生成 AI（ChatGPT / Claude など）… 下のプロンプト例を参考に

## AI への聞き方（プロンプト例）

そのままコピーして、`〇〇` を書き換えて使ってください。

**やり方を聞く**

```
HTML と CSS の初心者です。〇〇したいです。
コード例と、その説明を初心者向けに教えてください。
```

**うまく動かないとき**

```
次の HTML / CSS が思ったとおりに動きません。
- やりたいこと：〇〇
- 今のコード：
（ここにコードを貼る）
原因と、直したコード、その理由を初心者向けに教えてください。
```

**エラーの意味を聞く**

```
次のエラーの意味と直し方を、初心者向けにやさしく教えてください。
（ここにエラーメッセージを貼る）
```

### うまく聞くコツ

- 最初に **前提**（初心者・HTML/CSS）と **やりたいこと** を書く
- **今のコード** と **期待する結果** を貼る
- 1 回で完璧を狙わず、「もっとやさしく」「別のやり方は？」と続けて聞く

## 注意（とても大事）

- AI の答えは **まちがうこともある**。必ず **自分で動かして確認** し、できれば MDN で裏を取る。
- **コピペだけ** だと身につかない。コードの意味を一行ずつ理解しながら使う（模写演習の注意と同じ）。
