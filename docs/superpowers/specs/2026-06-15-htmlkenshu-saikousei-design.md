# HTML 研修教材 抜本再構成 設計書

- 作成日: 2026-06-15
- 対象リポジトリ: htmlkenshu (ITSVAPP/htmlkenshu)
- 対象ブランチ: main（解答は answer ブランチ）

## 1. 背景と目的

新入社員（HTML 初学者）向けの学習教材。現状はフォルダ構成・内容に問題があり、初心者が「何を・どの順で・どこまで」やればよいか俯瞰できない。

本設計の目的は、**初心者が上から順にやれば一周できる「一本道」** へ抜本再構成し、内容を最新の Web 標準に更新すること。

### 現状の主要課題（棚卸し済み）

- **番号と学習順の逆転**: 導入である `90_その他演習/10_EラーニングによるHTML・CSSの導入` が番号上は最後（90）にある。
- **`00_内定者研修用` だけ役割が違う**: 他（10/20/30）は統一フォーマットの「演習」だが、ここは「参考＋サンプル集」。名称「内定者研修用」も実態（新入社員/初学者向け）と不一致。トップ README は 00 を「開発者ガイド」と案内しており意味が通らない。
- **`01_調べるときの参考/参考サイト.txt`**: `.txt` 形式でリンク非クリック・非レンダリング。`http://www.htmq.com`、`https://developer.mozilla.org/ja/docs/Web/Reference`（リンク切れの可能性大）、個人ブログ複数（リンク腐敗リスク）。
- **`02_サンプル`**: HTML5 で廃止された `<font>` / `<center>` / `align` 属性 / 廃止 iframe 属性（`frameborder`/`scrolling`/`allowTransparency`）を教えている。本文に現存しない紙テキストへの参照（`テキスト27P参照` 等）が残る。題材も陳腐（Twitter＝現 X / Facebook プラグイン埋め込み、文字化け対応）。
- **`03_自己紹介ページサンプル`**: `Thumbs.db`（Windows のゴミ）がコミット済み。7 サンプルに説明がなく、初心者がどれを参考にすべきか不明。
- **解答が answer ブランチ運用**: 初心者にはブランチ切替がハードル。
- **全体カリキュラム表（俯瞰）が無い**。

## 2. 決定事項（ユーザー選択）

- **見直し規模**: 抜本的に再構成（学習順に番号を振り直し、`00_内定者研修用`・`90_その他演習` を解体・統合）。
- **解答の渡し方**: ヒント同梱＋解答はブランチ。各課題 README に「考え方・ヒント・解説」を文章で同梱（ブランチ操作なしで詰まらない）。完全な解答コードは従来どおり answer ブランチに残し、新構成へ同期する。

## 3. 最終形（目標フォルダ構成）

```
00_はじめに/
  README.md          … 環境構築(VSCode/拡張/Live Server) ＋ 学習マップ(全体カリキュラム表)
01_導入_Eラーニング/    ← 旧 90/10。最初の一歩
02_自己紹介ページ/
  README.md          … 演習手順(旧 90/20)
  サンプル/           ← 旧 00/03 の7サンプル＋各サンプルに「学べること/難易度」
03_模写コーディング/     ← 旧 90/30。自己紹介の次・CSSフォームの前に配置
04_CSSフォーム/        ← 旧 10（中身維持・入口README整備）
05_JSフォーム/         ← 旧 20
06_WebAPI/            ← 旧 30
99_リファレンス_調べ方/
  README.md          … 逆引き表（初心者が困ったこと → アンサーのURL or サンプルコードの場所）。旧 00/01 を刷新
  サンプル/           ← 困りごと単位の、非常にシンプルで分かりやすいHTML。旧 00/02 を刷新・現代化
```

初心者は **00 → 01 → … → 06 を上から順** にやれば一周でき、困ったら 99 を引く、という一本道になる。学習順の意図は「導入(01) → 自分のページを作る(02) → 既存サイトを模写して総合力をつける(03) → フォーム/JS/API の個別演習(04〜06)」。

## 4. 変更詳細（ワークストリーム）

### WS1. フォルダ再編（リネーム/移動）
- `git mv` で番号・名前を新構成へ（履歴温存）。`00_内定者研修用`・`90_その他演習` を解体。
- 意味のある単位でコミット分割。

### WS2. リンク・参照の更新
- トップ README の「00_内定者研修用＝開発者ガイド」リンク、資材ダウンロードリンク。
- 旧 `90/20` README の参照パス（→ `02_自己紹介ページ/サンプル`）。
- 各 README 内の相対リンク・画像パス。

### WS3. 参考集の刷新（`99_リファレンス_調べ方`）
- `.txt` → `README.md`。**逆引き表**に再編。列は2軸:
  - **初心者が困ったこと**（例: 「文字を中央に寄せたい」「画像を入れたい」「リンクを貼りたい」「表を作りたい」）
  - **アンサー**: MDN 等の **公式 URL**、または配下 `サンプル/` の **該当サンプルコードの場所**
- 困りごと単位で、外部 URL と配下サンプルを対応づける（どちらか一方でも可）。
- MDN 等の公式・一次情報中心。全リンクのリンク切れ確認。

### WS4. サンプルの現代化
- **自己紹介サンプル（旧 00/03）** → `02_自己紹介ページ/サンプル/`。各サンプルに「学べること/難易度」を記した README。`Thumbs.db` 除去。同梱フォントのライセンス確認。
- **参考サンプル（旧 00/02 を刷新）** → `99_リファレンス_調べ方/サンプル/`。
  - **1ファイル＝1テーマ（困りごと単位）** に再編し、**初心者が一目で分かる、非常にシンプルで短い HTML** に書き直す（余計な要素を入れない／コメントで要点を一言）。
  - ファイル名を困りごとに対応させ、WS3 の逆引き表から参照できるようにする。
  - 廃止タグ（`<font>`/`<center>`/`align`/廃止 iframe 属性）を CSS/HTML5 に書き換え。
  - 本文の「テキスト XXP 参照」を除去し、自己完結した内容に。
  - 陳腐な回（文字化け対応、旧 SNS 埋め込み）は廃止。文字寄せ・画像・リンク・表・背景・favicon・`:hover`・Google Maps 等は維持・現代化・動作検証。

### WS5. README 雛形の統一とカリキュラム表
- 全ステップ共通の README 雛形（目的 / 前提 / 手順 / ゴール完成イメージ / 所要目安）。
- トップ README にカリキュラム表（順序・目安時間・前提・ゴール）。

### WS6. 課題への解説・ヒント同梱
- 各課題 README に「考え方・ヒント・解説」セクションを追加（完全な解答コードは載せない）。

### WS7. 雑務整理
- `Thumbs.db` 削除、`.gitignore` に `Thumbs.db` / `.DS_Store` を追加。
- `.url`（Windows 専用ショートカット）→ README 内リンク化。

### WS8. answer ブランチ同期
- main の新構成と同一の番号・名前へ answer ブランチを再編。
- answer 側のみ存在する `00_内定者研修用/00_説明資料` の移設先を決定（→ 該当ステップ配下）。
- main 再編とセットで実施し、構成の乖離を防ぐ。

## 5. 互換性・リスク

- **外部リンク切れ**: フォルダ名変更で既存ブックマーク/共有リンクが切れる → トップ README にリリースノート的な案内を記載。
- **answer 乖離**: WS8 を必須化し、main 再編と同時に実施。
- **大量 git mv**: コミットを意味単位に分割し、履歴の追跡性を確保。

## 6. 非対象（YAGNI）

- 既存演習（CSS/JS/WebAPI）の課題内容そのものの作り直しはしない（番号・入口 README 整備のみ）。
- 新規演習の追加はしない。
- ビルドツール / フレームワーク導入はしない（素の HTML/CSS/JS のまま）。

## 7. 受け入れ基準

- 初心者が 00 → 06 を上から順に実施でき、各ステップに目的・手順・ゴール像がある。
- 廃止タグ・「テキスト XXP 参照」・リンク切れ・`Thumbs.db` が存在しない。
- 99 を引けば「やりたいこと」から調べられる。
- main と answer のフォルダ構成が一致している。

## 8. 実施順序（概略）

1. WS7（軽い掃除）→ WS1（再編）→ WS2（リンク）… 構造を先に固める。
2. WS5 / WS6（README 整備）→ WS3 / WS4（内容刷新）。
3. WS8（answer 同期）。
